- 使用 CREATE 创建合约时：  
    **新合约地址 = keccak256(sender, nonce)**  
    👉 其中 **nonce 是动态变化的**（每次部署、新交易都会导致变化）。
    
- 使用 CREATE2 创建合约时：  
    **新合约地址 = keccak256(0xFF, creator, salt, keccak256(initcode))**  
    👉 不再使用 nonce。所以可以预测

nonce = 一个账户已经“做过几次动作”的计数器
这里的 **nonce 指的是“创建者的 nonce”**。
比如工厂合约连续用 `new` 创建 3 个子合约，你觉得它的 nonce 变化是：每次创建一个子合约，nonce 加 1

salt 是 **创建者自己随便给的一个 bytes32 值**。
PairFactory 例子里，salt 是这样确定的：
`bytes32 salt = keccak256(abi.encodePacked(token0, token1));`

initcode = 编译出来的 Pair 合约的 creation bytecode（固定，编译后就确定）
